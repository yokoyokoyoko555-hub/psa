# PRICING — 料金体系リニューアル 設計書（PSA日本）

> Codex / Claude Code 共有用の自己完結ドキュメント。関連: [ADR-0015](DECISIONS.md) / 既存料金は [ADR-0010](DECISIONS.md)/[ADR-0011](DECISIONS.md)。
> 状態: **設計確定（実装前・要確認2点あり §9）** / 最終更新: 2026-06-23 / 対象: **PSA日本(PSA_JP)のみ**（US は据え置き）

---

## 1. 目的
管理画面（ADMIN 料金設定）と顧客画面の料金を、実運用の体系に合わせて更新する。
**鑑定料は現状維持**。送料・保険を「申告価格帯×枚数帯のマトリクス」へ、加えて**代理入力料金**・**事務手数料**を追加し、すべて**サービスレベル単位（=1申込=1サービスレベル）で請求**できるようにする。

## 2. 確定方針（2026-06-23 ユーザー承認済み）
| 項目 | 決定 |
|------|------|
| 鑑定料 | **既存のまま**（`ServicePrice.pricePerCard` × 枚数、レベル×地域） |
| 送料・保険 | **合算で1つのマトリクス**（申告価格合計帯 × 枚数帯 → 金額）。顧客には「送料・保険料」1行で表示。26枚以上は加算ルールあり（§5） |
| 代理入力料金 | **¥/枚・サービスレベルごと**（例 990円/枚 → 10枚=9,900円）。代理入力(STORE)時のみ加算 |
| 事務手数料 | **¥/申込・サービスレベルごと**（申込に1回。申込=1サービスレベル） |
| 範囲 | **PSA日本のみ**。PSA US は現状据え置き |
| 税 | 一律10%（変更なし） |

## 3. 料金構成（PSA日本・1申込）
```
請求額 = 鑑定料(pricePerCard×枚数)
       + 送料・保険(マトリクス: 申告合計帯×枚数帯)
       + 代理入力料金(¥/枚×枚数)   ※代理入力(STORE)時のみ
       + 事務手数料(¥/申込)
       + 消費税10%
```
> 申込は1サービスレベルにつき1申込。複数レベルに出す場合は申込が分かれる。

## 4. データモデル
### 4.1 `ServicePrice`（既存・レベル×地域）に追加
| フィールド | 内容 |
|-----------|------|
| `pricePerCard` | 鑑定料/枚（既存・不変） |
| `agencyFee` | **代理入力料金 ¥/枚**（既存フィールドを流用・UIラベル変更）。STORE時のみ加算 |
| `maxDeclaredValue` | 申告価格上限（既存・不変） |
| `handlingFee` Int @default(0) | **事務手数料 ¥/申込（新規追加）** |

### 4.2 新規 `ShippingInsuranceRate`（送料・保険 合算マトリクス）
| フィールド | 内容 |
|-----------|------|
| region | ServiceRegion（PSA_JP） |
| minValue / maxValue Int? | 申告価格**合計**の帯（maxValue=null は上限なし） |
| qtyMin / qtyMax Int? | 枚数帯（例 1-8 / 9-25 / 26-）。qtyMax=null は上限なし |
| fee Int | セル金額（1-8 / 9-25 帯の定額。26+帯では基準額） |
| perCardSurcharge Int @default(0) | 26+帯の¥/枚加算（25 or 50。通常帯は0） |
| sortOrder / isActive / updatedAt / updatedBy | 運用 |

> 既存 `ShippingRule`（返却方法×申込金額帯）・`InsuranceRule`（申告額帯）は **PSA_JP では本マトリクスに置換（廃止）**。テーブルは残置可（US用に当面温存 or 後で整理）。

## 5. 計算式（`fee-calculator` 改修）
1. 鑑定料 = `pricePerCard × 枚数`（不変）。
2. 送料・保険 = `ShippingInsuranceRate` から **(申告合計が[minValue,maxValue]) かつ (枚数が[qtyMin,qtyMax])** の行を取得。
   - 通常帯（1-8 / 9-25）: `fee`。
   - 26+帯: **`fee(基準額) + perCardSurcharge × ?枚`**（§9 要確認: 全枚数 or 25超過分）。
3. 代理入力料金 = STORE時のみ `agencyFee × 枚数`。
4. 事務手数料 = `handlingFee`（申込に1回）。
5. 小計 → 税10% → 合計。

## 6. 管理画面（ADMIN 料金設定）
- サービスレベルごとに **鑑定料(¥/枚) / 代理入力料金(¥/枚) / 事務手数料(¥/申込) / 申告上限** を編集（既存フォーム拡張＋新規作成対応）。
- **送料・保険マトリクス**のエディタを新設（申告価格帯×枚数帯の表を行追加/編集。26+帯の加算単価も設定）。
- 既存の送料/保険フォームは PSA_JP では新マトリクスへ置換。

## 7. 顧客画面
- 申込フォーム/確認/詳細の内訳に **鑑定料・送料・保険料・代理入力料金(代理時)・事務手数料・消費税** を表示。
- `FeeBreakdown` に `handlingFee` を追加し、送料・保険は合算1行（`shippingInsuranceFee`）。

## 8. 範囲・非対象
- 本リニューアルは **PSA_JP のみ**。PSA_US は既存ロジック据え置き（将来別途）。
- 鑑定料テーブル・税率・ステータス遷移は変更しない。

## 9. 確定した金額ルール（2026-06-23）
1. **26枚以上**: `送料保険 = 9-25帯の金額 + 加算単価 × (枚数 − 25)`。加算単価 = 申告合計 ≤¥10,000,000 → **¥25/枚**、超 → **¥50/枚**。
   - 実装: 26+帯の行に `fee`(=9-25帯と同額の基準) と `perCardSurcharge`(25 or 50) を保持し、`fee + perCardSurcharge × (枚数 − 25)`。
2. **店頭受取でもマトリクス満額**を請求（送料・保険は返却方法に依存しない＝`ShippingInsuranceRate` は returnMethod を持たない）。
